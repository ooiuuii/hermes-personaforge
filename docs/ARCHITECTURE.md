# Architecture

Hermes PersonaForge separates the business layer from the local character runtime layer.

## Flow

```text
Paid persona-pack order
  -> payment reconciliation
  -> Hermes margin gate
  -> persona license manifest
  -> runtime launch trace
  -> safety / asset boundary report
  -> audit ledger
  -> proof manifest
```

## Components

| Component | Responsibility |
| --- | --- |
| `artifacts/input/persona-commerce-order.json` | Redacted paid-order input generated by the proof script |
| `artifacts/proof/payment_reconciliation.json` | Entitlement proof from a paid store order |
| `artifacts/proof/persona_margin_gate.json` | Revenue, cost, profit, and unlock/hold decision |
| `artifacts/proof/persona_license_manifest.json` | License id, pack id, manifest hash, capabilities, and usage scope |
| `artifacts/proof/runtime_launch_trace.json` | Deterministic trace for voice, vision, browser context, and sprite switching |
| `artifacts/proof/safety_redaction_report.json` | Proof that local character assets are not redistributed |
| `artifacts/proof/persona_audit_ledger.json` | End-to-end timeline from paid order to runtime session |
| `scripts/generate-proof-pack.mjs` | Regenerates all verifier artifacts |
| `scripts/verify-proof-pack.mjs` | Checks payment, margin, license, runtime, safety, and manifest claims |
| `src/App.tsx` | Interactive workbench |
| `src/CinematicTrailer.tsx` | Video capture scenes |
| `scripts/record-full-demo.mjs` | Records the local demo into MP4 segments |

## Runtime Boundary

Public repo:

- persona product workflow,
- payment/margin/license proof,
- runtime launch trace,
- safety policy,
- audit ledger,
- demo UI.

Private local runtime:

- character pack files,
- voice checkpoints,
- sprite sheets,
- local runtime code,
- model weights,
- cookies or credentials.

This lets the video show the rich companion experience while the repository remains clean and legally safer.
