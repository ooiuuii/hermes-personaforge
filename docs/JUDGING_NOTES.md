# Judging Notes

## Usefulness

PersonaForge is a concrete business tool for a near-term creator economy product: paid AI persona packs. It handles the hard operational parts around the emotional demo:

- payment proof,
- profitable delivery decision,
- license manifest,
- runtime activation,
- refund/hold path,
- redaction and asset boundary,
- audit evidence.

The larger market is not just one character pack. PersonaForge can become a marketplace and consignment layer where licensed IP owners and independent agent artists sell persona packs with rights, revenue share, delivery, and proof handled by Hermes.

## Viability

The project builds on a real commerce rail already proven in the previous submission. The new surface is different: the paid product is a persona-pack experience rather than a generic service or order workflow.

The business model has three viable supplier paths:

- licensed IP owners releasing official persona packs,
- agent artists creating original characters with voice, memory, tools, scenes, and demo videos,
- custom commissioned personas for customers or brands.

Hermes is useful here because it can enforce the rights manifest, margin gate, refund/hold paths, entitlement delivery, and revenue split proof.

The demo can be run without secrets:

```powershell
npm install
npm test
npm run build
```

`npm test` regenerates and verifies the proof pack.

## Presentation

The video should lead with the emotional moment:

1. A user buys a persona pack.
2. Hermes verifies the paid order and unlocks a license.
3. The runtime character reads a page about itself.
4. The character reacts with voice and a shy expression.
5. The proof pack shows that this was a controlled business operation, not just a character toy.
6. The marketplace model explains how the same loop scales to agent artists and licensed IP packs.
